Обучение Git

  • Git – популярная система для хранения программного кода, относящаяся к категории систем контроля версий

    Запомнить! CVS (Control Version System) - Система контроля версий, позволняет вернуться к определённой версии файла или определить, кем и когда были сделаны изменения в файле.

    Запомнить! Репозиторий Git – хранилище для истории разработки проекта, которое содержит все версии используемых файлов.

    Запомнить! Коммит (commit) - фиксация состояния проекта в репозитории Git

    Запомнить! Индекс Git – список файлов, изменения в которых система Git отслеживает. В индексе также предварительно сохраняются все изменённые файлы, которые будут зафиксированы в репозитории при следующем коммите

    Запомнить! Ветка Git – изолированный независимый поток разработки, в котором можно делать коммиты, недоступные для других веток

    Система контроля версий позволяет:

    1. Добавлять новый функционал.
    2. Исправлять ошибки.
    3. Работать в команде.

  • Каждый Git-проект состоит из трёх областей: рабочий каталог, индекс и репозиторий.

    1. HEAD - текущий слепок на момент последнего коммита (это слепок последних закомиченных изменений)
    2. Рабочий каталог - текущие файлы над которыми мы работаем. Когда мы создаем новые файлы или изменяем уже существующие, то эти файлы сохраняются в рабочем каталоге. Рабочий каталог не имеет отношения к папке .git.
    3. Индекс находится в папке .git и содержит информацию о том, какие именно файлы и в каком состоянии будут зафиксированы при следующем коммите.

      Изменим файл style.css и пропишем git add .

      Тем самым добавим файл в группу индекс (индекс содержит файлы готовые для коммита).

    Обработка файлов в Git происходит следующим образом:

    1. Мы редактируем файл в рабочем каталоге. У такого файла будет состояние "Изменен" (modified).
    2. Индексируем изменённый файл, добавляя его в индекс командой git add. Статус такого файла – "Индексирован" (staged)
    3. Сохраняем изменения в репозитории, выполняя коммит с помощью git commit. Статус такого файла – "Зафиксирован" (fixed).

    Посмотреть содержимое файла в консоле можно с помощью команды cat ...

    Что-бы откатиться к нужному коммиту существует команда git reset с параметрами --soft --hard

    git reset --soft коммит переместит указатель HEAD на нужный коммит, при этом рабочий каталог и индекс остаются прежними (соответствующими коммиту, с которого мы ушли). (Изменения которые произошли после коммита к которому хотим перейти останутся, можно будет открыть нужный файл, произвести в нём изменения, и затем закоммитить, и будет создан следующий коммит).

    git reset --hard коммит позволяет полностью отменить все изменения, которые были сделаны после определённого коммита. (удаляет файлы и полностью возрощает состояние проекта к текущему коммиту к которому мы хотим откатится).

    git reset --soft
